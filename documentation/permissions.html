FBC:Permissions template tag

Permissions from Facebook site for use on your EE site are granted during Login process or Set_Permissions process. This template tag allows you to check which Facebook permissions the logged in user has granted.

Facebook allows the following permissions to be request from users without needing manual review by Facebook:

  • public_profile
  • email
  • user_friends

These are the main permissions you'll need as well, with the exception of publish_actions, which allows your users to publish data from your site onto their Facebook profile.

You can view the full list of permissions in the Facebook Permissions Documentation. Use the names mentioned in the User permission column.

If you intend on using/requesting any information other than public_profile, email and user_friends, you'll need to obtain permission manually from Facebook. For example, if you wish to allow users to publish content such as EE comments to the users' Facebook profile (publish_actions), you'll need to manually request permission from Facebook. You can learn more about How to Submit for Login Review (Facebook Developer Documentation) here.

To check against permissions granted, format your conditionals like this, where PERMISSION_NAME is the name of the permission in the Facebook Permissions Documentation):

{if fbc_allow_PERMISSION_NAME == "y"}{/if}

Example

Here’s an example of how you would determine some permissions of a logged in user:

<h3>Permissions I have granted for {site_name}:</h3>
<ul>
{exp:fbc:permissions}
    <li>
        Public Profile?
        <b>{if fbc_allow_public_profile == "y"}Yes{if:else}No{/if}</b>
    </li>
    <li>
        Email Address?
        <b>{if fbc_allow_email == "y"}Yes{if:else}No{/if}</b>
    </li>
    <li>
        User Friends List?
        <b>{if fbc_allow_user_friends == "y"}Yes{if:else}No{/if}</b>
    </li>
    <li>
        Allow Publishing to Facebook?
        <b>{if fbc_facebook_allow_publish_actions == "y"}Yes{if:else}No{/if}</b>
    </li>
{/exp:fbc:permissions}
</ul>
